# 後台按鈕與操作規範

本文件適用於主辦方、攤主與管理員後台，僅記錄按鈕的出現條件、用途與操作流程。

## 共用操作原則

1. **查看、編輯**：進入同一後台內的頁面，不顯示確認視窗。
2. **需要填寫資料**：使用共用表單 Modal 或 SweetAlert 表單，例如補件、退款、下架或駁回原因。
3. **會改變狀態**：先顯示確認 Alert，API 成功後才顯示結果 Alert。
4. **外部或跨系統內容**：才開新分頁；同一後台功能維持原分頁。
5. **危險操作**：停用、駁回、下架等操作必須清楚說明影響，不能只以按鈕顏色提示。
6. **API 失敗**：保留使用者已填內容，顯示失敗原因，不得先更新畫面狀態。
7. **按鈕名稱**：需明確描述實際操作，例如下架案件使用「下架審核」，不以泛稱「審核」代替。

標準流程：`填寫（如需要） → 確認 → API → 成功／失敗提示`

## 攤主可報名活動

- 可報名活動列表與首頁活動卡只顯示目前在報名期間內的活動，不顯示尚未開放或已截止活動。
- 活動仍有攤位時標示「報名中」；所有活動日皆無剩餘攤位時標示「已額滿」。
- 「已額滿」活動仍可顯示供攤主查看，但不可送出報名；既有報名與歷史活動由報名紀錄頁提供，不混入可報名活動列表。

## 主辦方後台

### 活動管理

| 狀態 | 按鈕 | 操作方式 |
| --- | --- | --- |
| 草稿 | 編輯、送出審核 | 編輯進頁面；送審用確認 Alert |
| 待審核 | 查看詳情、撤回申請 | 查看進頁面；撤回用確認 Alert |
| 補件中 | 編輯、重新送審 | 編輯進入活動修改頁並顯示補件原因；重新送審用確認 Alert |
| 地圖建置中 | 查看詳情 | 進頁面 |
| 待發布 | 查看詳情、發布活動 | 查看進頁面；發布用確認 Alert |
| 報名中／已額滿 | 查看詳情、下架活動 | 查看進頁面；填下架原因後確認 |
| 下架申請中 | 查看詳情 | 進頁面 |
| 已發布／最終確認／進行中／已結束／已下架 | 查看詳情 | 進頁面 |

> 「補件中」的活動在管理列表與活動詳情皆顯示「編輯、重新送審」；進入編輯頁後會持續顯示補件原因。
>
> 主辦方送出下架申請後，前端必須重新載入 API 資料並顯示「下架申請中」，不可在 API 成功前自行改寫本地狀態；下架申請不提供撤回操作。
>
> 下架審核期間活動仍公開，攤主既有報名流程維持正常。攤主報名詳情需顯示等待審核提示；核准下架後改顯示活動已下架及後續退款流程提示，但前端不自行建立退款。
>
> 活動攤位分區名稱固定選擇 `A 區` 至 `Z 區`，同一活動不可重複且最多 26 區；分區英文字母會作為後續攤位編號前綴。

> 管理員完成地圖建置後，後端會依分區數量自動建立 `A01`、`A02`、`B01` 等攤位；活動發布前若數量不足，前端應顯示後端回傳的「目前已建立 X / Y 個攤位」，不可只顯示籠統錯誤。

### 報名管理

| 狀態 | 按鈕 | 操作方式 |
| --- | --- | --- |
| 待審核 | 查看、審核 | 查看進頁面；審核結果用 Modal／確認 Alert |
| 待付款／待選位 | 查看 | 進頁面 |
| 報名完成 | 查看、退還保證金 | 查看進頁面；退還用確認 Alert |
| 退款申請中 | 查看、前往確認退款 | 查看進頁面；退款處理進收款詳情 |
| 退款處理中／已退款／已取消 | 查看、退款資訊 | 進收款詳情 |
| 保證金已退還 | 查看、保證金退還資訊 | 進收款詳情 |
| 審核未通過 | 查看 | 進頁面 |

### 收款、攤位、設備與帳務管理

| 管理頁／狀態 | 按鈕 | 操作方式 |
| --- | --- | --- |
| 收款管理／所有狀態 | 查看收款詳情 | 進頁面 |
| 收款管理／退款申請中 | 確認退款 | 在收款詳情填寫或確認後送出 |
| 攤位管理 | 查看詳情、查看攤位地圖 | 進頁面 |
| 設備管理 | 查看 | 進頁面 |
| 帳務管理 | 查看 | 進帳務詳情 |
| 帳務詳情 | 查看收款詳情 | 進收款詳情 |

## 管理員後台

### 活動管理

| 狀態 | 按鈕 | 操作方式 |
| --- | --- | --- |
| 待審核／補件完成 | 審核、查看 | 兩者皆進活動詳情，不在列表直接改變狀態；審核置左、查看置右 |
| 待審核／補件完成詳情 | 要求補件、審核通過 | 補件先填原因再確認；通過使用確認 Alert |
| 地圖建置中 | 審核、查看／地圖建置完成 | 列表的審核與查看皆進詳情；完成使用確認 Alert |
| 下架申請中 | 下架審核、查看 | 兩者皆進詳情；下架審核使用 Modal 選擇同意或不同意，不同意時必填說明 |
| 下架申請中詳情 | 下架審核 | 使用 Modal 選擇同意或不同意；不同意時必填說明 |
| 其他狀態 | 查看 | 進活動詳情，不顯示狀態變更按鈕 |

### 使用者管理

| 狀態／頁面 | 按鈕 | 操作方式 |
| --- | --- | --- |
| 使用者列表 | 查看 | 依角色進入主辦方或攤主詳情 |
| 正常帳號 | 停用帳號 | 顯示影響說明與確認 Alert；API 成功後更新狀態 |
| 已停用帳號 | 恢復帳號 | 顯示確認 Alert；API 成功後更新狀態 |
| 主辦方活動紀錄 | 查看 | 進活動詳情 |
| 攤主報名紀錄 | 查看 | 進報名或活動相關詳情 |

### 首頁、通知與操作紀錄

| 管理頁 | 操作方式 |
| --- | --- |
| 首頁待處理事項 | 點擊卡片進對應管理列表或待審核內容 |
| 最新通知／通知中心 | 點擊通知進對應詳情；已讀狀態由通知元件統一管理 |
| 操作紀錄 | 僅供查詢與篩選，不提供修改或刪除操作 |

## 攤主後台

### 我的報名紀錄與報名詳細

| 狀態 | 按鈕 | 操作方式 |
| --- | --- | --- |
| 待審核 | 取消報名、查看 | 兩者皆先進報名詳細；取消報名時顯示影響說明與確認 Alert |
| 待付款 | 查看、前往付款、取消報名 | 先進報名詳細；由詳細頁前往付款，取消報名時顯示確認 Alert |
| 待選位 | 查看、選擇攤位、退款申請 | 先進報名詳細；由詳細頁選位，退款申請需填寫原因並再次確認 |
| 報名完成 | 查看 | 進報名詳細，可查看攤位與費用資訊 |
| 退款申請中／退款處理中／已退款 | 查看 | 進報名詳細查看退款進度，不在列表直接改變狀態 |
| 保證金已退還／已取消／歷史紀錄 | 查看 | 進報名詳細，僅提供紀錄查詢 |

### 我的攤位、付款、首頁與通知

| 管理頁 | 按鈕／操作 | 操作方式 |
| --- | --- | --- |
| 我的攤位 | 新增、編輯、刪除商品 | 在同頁編輯；刪除商品需先顯示確認 Alert |
| 我的攤位 | 儲存變更 | 驗證表單後呼叫 API；成功後顯示結果 Alert，失敗時保留內容 |
| 付款頁 | 前往信用卡付款 | 顯示付款資訊後進金流；金流屬外部系統時才開新分頁 |
| 首頁摘要卡片 | 點擊卡片 | 進對應的報名紀錄、攤位或通知頁面 |
| 最新通知／通知中心 | 點擊通知 | 進對應詳情；已讀狀態由通知元件統一管理 |
